วันนี้เรียนเรื่อง export เพื่อนำไปพํฒนาต่อตอนเขียน framework
โดยที่ export ได้ 2 แบบคือ export default และ export name
ต่างกันตรงที่เวลา import จากไฟล์อื่น จะมีการเข้าถึงค่าไม่เหมือนกัน

1. export default จะใช้ได้แค่ครั้งเดียวต่อไฟล์นั้นๆ ไม่สามารถส่งหลายรอบได้
เวลา import จะอยู่ในรูปนี้ :
import sayHello from "./greeting.js"
โดยเราจะเปลี่ยนชื่อสิ่งที่ import มาเป็นอะไรก็ได้ เพราะมันชี้ไปที่ไฟล์นั้นๆอยู่แล้ว ยังไงมันก็จะหากันเจอ

2. export name จะส่งกี่ค่าก็ได้ โดยที่ต้องมีปีกกา ครอบตอน import
เวลา import จะอยู่ในรูปนี้ :
import {sayHello,helloSay} from "./greeting.js"

โดยเราสามารถใช้ ทั้งสอบแบบพร้อมกันได้
ยกตัวอย่างเช่น :
import course, {courseDetail} from "./data.js"

นอกจากนี้วันนี้ก็ได้เห็น async function สรุปคร่าวๆมันเป็น function ที่จะคืนค่าเป็น promise ... เสมอ
เหมือนเป็นการรอ ค่าๆนึงก่อนโดยใช้ await และ then ในการ pending 
ส่วนมากจะใช้ตอนเรา fetch data จาก backend มาแล้วต้องการรอให้ข้อมูลมาครบก่อนค่อยทำ process ถัดไป
ซึ่งผมจะฝึกและทำความเช้าใจ async function นี้ให้มากขึ้นในสัปดาห์ถัดไป
